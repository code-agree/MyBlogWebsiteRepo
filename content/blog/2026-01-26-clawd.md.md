# 解决 macOS 上 clawdbot 进程无法终止的问题

## 问题现象

在使用 `kill` 命令终止 `clawdbot` 和 `clawdbot-gateway` 进程时，发现它们会立即重启，PID 不断变化但进程始终存在。

```bash
$ ps -ef | grep clawd
  501  1494     1   0  1:24上午 ??  0:00.11 clawdbot    
  501  1495  1494   0  1:24上午 ??  0:03.00 clawdbot-gateway

$ kill -9 1494 1495

# 进程又出现了，只是 PID 变了
$ ps -ef | grep clawd
  501  1667     1   0  1:26上午 ??  0:00.13 clawdbot    
  501  1668  1667   0  1:26上午 ??  0:03.89 clawdbot-gateway
```

## 问题分析

从进程信息可以看出几个关键点：

1. **父进程为 1（launchd）**：`clawdbot` 的 PPID 是 1，说明它由系统的 `launchd` 进程管理
2. **进程层级关系**：`clawdbot-gateway` (PID 1668) 是 `clawdbot` (PID 1667) 的子进程
3. **自动重启机制**：每次 kill 后进程立即重启，这是典型的 LaunchAgent/LaunchDaemon 行为

在 macOS 中，LaunchAgent 是一种系统服务管理机制，用于自动启动和维护后台进程。当被管理的进程意外终止时，launchd 会自动重启它。

## 解决步骤

### 第一步：定位 LaunchAgent 服务

使用 `launchctl list` 查看系统中运行的所有服务：

```bash
$ launchctl list | grep -i clawd
1715 1 com.clawdbot.gateway
```

输出说明：
- `1715`：进程 PID
- `1`：最后退出状态码
- `com.clawdbot.gateway`：服务标识符（Label）

找到了！原来是名为 `com.clawdbot.gateway` 的 LaunchAgent 在管理这些进程。

### 第二步：卸载 LaunchAgent 服务

使用 `launchctl bootout` 命令卸载服务：

```bash
$ launchctl bootout gui/$(id -u)/com.clawdbot.gateway
```

命令解释：
- `gui/$(id -u)`：当前用户的 GUI 会话域
- `com.clawdbot.gateway`：要卸载的服务名

### 第三步：删除配置文件

查找并删除 LaunchAgent 的配置文件（.plist），防止系统重启后再次加载：

```bash
# 查找配置文件
$ find ~/Library/LaunchAgents /Library/LaunchAgents -name "*clawdbot*"
/Users/username/Library/LaunchAgents/com.clawdbot.gateway.plist

# 删除配置文件
$ rm ~/Library/LaunchAgents/com.clawdbot.gateway.plist
```

### 第四步：验证结果

```bash
# 确认服务已移除
$ launchctl list | grep -i clawd
(无输出)

# 确认进程已停止
$ ps -ef | grep clawd
  501  1694   762   0  1:26上午 ttys000  0:00.00 grep clawd
(仅显示 grep 命令本身)
```

至此，问题彻底解决！

## 解决方案总结

处理 macOS 上"杀不死"的进程，核心思路是：

1. **找到根源**：通过 `launchctl list` 定位管理进程的服务
2. **卸载服务**：使用 `launchctl bootout` 停止服务
3. **删除配置**：移除 .plist 文件防止复发
4. **验证结果**：确认进程和服务都已清除

## 相关知识

### LaunchAgent 配置文件位置

macOS 中 LaunchAgent 配置文件通常位于以下目录：

| 路径 | 作用范围 | 权限 |
|------|---------|------|
| `~/Library/LaunchAgents/` | 当前用户 | 用户 |
| `/Library/LaunchAgents/` | 所有用户 | root |
| `/Library/LaunchDaemons/` | 系统守护进程 | root |

### 为什么不能直接 kill

直接使用 `kill` 命令只是终止进程，但 LaunchAgent 服务仍在运行。launchd 会根据配置文件（.plist）中的设置自动重启进程，这就是为什么进程"杀不死"的原因。

正确的做法是先卸载服务，切断 launchd 的自动重启机制，然后再清理配置文件。
